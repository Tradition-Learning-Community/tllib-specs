# Math contract entry preparation

This package prepares a limited candidate wave for mathematical-contract authoring. It does not create or approve mathematical contracts, executable oracles, IR, or implementation.

## Outcome

- READY_FOR_MATH_CONTRACT: 50
- READY_WITH_CONTRACT_RESERVATIONS: 16
- REQUIRES_TARGETED_SOURCE_REVIEW: 0
- REQUIRES_SCIENTIFIC_DECISION: 0
- NOT_CONTRACTABLE_AS_FEATURE: 0
- CONTRACT_WAVE_1: 5

The distinction between `oracle_basis` and `test_oracle` is normative for this package: cited scientific material supports later oracle construction, while executable test artifacts and reference values remain absent.

## Method

Only the 66 `retained_with_reservations` features were inspected. Behaviors, candidate inputs and outputs, object identifiers, and source ranges are copied from the revised catalog. Oracle bases classify the cited objects by existing object type. No dependency is inferred: the revised records declare no immediate feature dependencies, so each entry records `no_contract_dependency` with local scientific evidence.

## Classification rule

A small candidate citing at most four scientific objects is marked ready. A grouped candidate, or one whose revised estimated scope is not small, is marked ready with contract reservations so contract authoring can bound or decompose it without inventing content.

## Safeguards

All artifacts remain `candidate`. Scientific sources, revised registries, reviews, and decomposition reports are inputs only. Readiness for IR and implementation remains false.
